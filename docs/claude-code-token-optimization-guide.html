1. Quick Wins: Global Settings

Apply these settings to ~/.claude/settings.json for immediate token savings across all projects.

Recommended Configuration

{
  "model": "sonnet",
  "cleanupPeriodDays": 10,
  "env": {
    "DISABLE_NON_ESSENTIAL_MODEL_CALLS": "1",
    "ANTHROPIC_DEFAULT_HAIKU_MODEL": "claude-3-5-haiku-20241022"
  },
  "smallModelOverride": "haiku",
  "permissions": {
    "deny": [
      "Read(./.env)",
      "Read(./.env.*)",
      "Read(./node_modules/**)",
      "Read(./build/**)",
      "Read(./dist/**)",
      "Read(./.next/**)",
      "Read(./coverage/**)",
      "Read(./.git/**)"
    ]
  }
}

What Each Setting Does

Setting Purpose Impact
model: "sonnet" Default to Sonnet instead of Opus ~5x cost reduction per token
cleanupPeriodDays: 10 Auto-delete old sessions (default: 30) Reduces disk usage, faster startup
DISABLE_NON_ESSENTIAL_MODEL_CALLS Skips cosmetic/flavor text generation Fewer unnecessary API calls
smallModelOverride: "haiku" Use Haiku for background tasks Cheaper sub-agent operations
permissions.deny Block reading large directories Prevents accidental context bloat

Environment Variable Override (Optional)

Add to your shell profile to redirect any Opus requests to Sonnet:

# PowerShell profile (~\Documents\WindowsPowerShell\Microsoft.PowerShell_profile.ps1)
$env:ANTHROPIC_DEFAULT_OPUS_MODEL = "claude-sonnet-4-5-20250929"

# Bash/Zsh (~/.bashrc or ~/.zshrc)
export ANTHROPIC_DEFAULT_OPUS_MODEL="claude-sonnet-4-5-20250929"

2. CLAUDE.md Optimization

Your project's CLAUDE.md file is loaded into context at session start. Every token counts.

Tiered Documentation Architecture

Structure your documentation to minimize what gets loaded into context:

Tier Location Content Loaded
Tier 1: Critical CLAUDE.md Commands, conventions, patterns Always
Tier 2: Contextual /docs/*.md Architecture, API specs On demand
Tier 3: Reference External links Full documentation Never auto-loaded

CLAUDE.md Best Practices

Do:
  • Keep it under 500 lines
  • Use bullet points over prose
  • Include essential commands: build, test, lint
  • Document project-specific conventions
  • List critical file paths
Don't:
  • Duplicate README content
  • Include verbose explanations
  • Embed full API documentation
  • List every file in the project
  • Include historical context unless critical

Example Optimized CLAUDE.md Structure

# Project Name

## Commands
- `npm run dev` - Start dev server (port 3000)
- `npm test` - Run tests
- `npm run lint` - ESLint + Prettier

## Stack
- Next.js 14 + TypeScript
- Prisma + PostgreSQL
- Tailwind CSS

## Conventions
- Components in `src/components/`
- API routes in `src/app/api/`
- Use `cn()` for className merging

## Critical Patterns
- Auth: NextAuth with JWT
- State: React Query for server state
- Forms: React Hook Form + Zod

Result: ~150 tokens vs. 800+ for a verbose version (63% reduction).

3. Session Management Habits

Essential Commands

Command When to Use Effect
/clear Switching to unrelated task Clears conversation history
/compact Conversation getting long Summarizes and compresses context
/cost Check current session usage Shows token count and cost

When to Use /clear

  • After completing a distinct task
  • When switching between unrelated features
  • Before starting debugging (fresh context helps)
  • When context becomes polluted with failed approaches

When to Use /compact

  • After 15-20 back-and-forth exchanges
  • When you notice Claude repeating context
  • Before asking for a summary or plan
  • When responses start getting slower
Pro Tip: Run /cost periodically during long sessions. If you see unexpected spikes, it may be time to /clear or /compact.

4. Model Selection Strategy

Model Comparison

Model Best For Cost (relative)
Haiku Simple tasks, quick lookups, formatting $
Sonnet Most coding tasks, debugging, refactoring $$
Opus Complex architecture, nuanced decisions $$$$$

Recommended Default: Sonnet

Sonnet provides the best cost-to-quality ratio for typical development work. Haiku mistakes can cost more tokens to fix than just using Sonnet upfront.

When to Use Haiku

  • Simple file searches
  • Code formatting tasks
  • Generating boilerplate
  • Quick syntax questions

When to Escalate to Opus

  • System architecture decisions
  • Complex multi-file refactors
  • Subtle bug diagnosis
  • Performance optimization analysis
Avoid Opus for: Routine coding tasks. A $0.50 Sonnet session that takes two attempts is still cheaper than a single $2.50 Opus session.

5. Advanced: Session Hooks

Session hooks allow automatic context management based on project detection.

Session-Start Hook Example

Create .claude/hooks/session-start.sh:

#!/bin/bash
# Detect project type and load appropriate context

if [ -f "package.json" ]; then
    if grep -q "next" package.json; then
        echo "Next.js project detected"
        echo "Key files: next.config.js, app/, components/"
    elif grep -q "react" package.json; then
        echo "React project detected"
        echo "Key files: src/, public/"
    fi
fi

if [ -f "requirements.txt" ] || [ -f "pyproject.toml" ]; then
    echo "Python project detected"
fi

Benefits of Hooks

  • Automatic project-type detection
  • Load only relevant documentation
  • Set project-specific environment variables
  • Pre-warm common file paths
Token Savings: Well-configured hooks can reduce initial context by 40-60% by loading only what's relevant.

6. Monitoring & Tracking

Built-in: /cost Command

Quick check of current session usage:

/cost

Shows: Total tokens, input/output breakdown, estimated cost.

ccusage: Detailed Analytics

ccusage is a community tool (9.2k+ stars) for detailed usage analysis:

# Install and run
npx ccusage@latest

# View daily breakdown
npx ccusage daily

# View by billing window (5-hour blocks)
npx ccusage blocks

# Live monitoring dashboard
npx ccusage blocks --live

# Filter by date range
npx ccusage daily --since 20250101 --until 20250131

# Export as JSON
npx ccusage daily --json

ccusage Report Types

Report Command Use Case
Daily ccusage daily Day-by-day usage overview
Monthly ccusage monthly Budget tracking
Session ccusage session Per-conversation breakdown
Blocks ccusage blocks Claude's 5-hour billing windows

Pro Tips for Cost Control

  1. Weekly reviews: Run ccusage monthly to spot trends
  2. Project attribution: Use --instances to see per-project costs
  3. Model breakdown: Use --breakdown to see cost by model
  4. Set alerts: Check daily usage before starting intensive work
Expected Savings: Following all practices in this guide typically yields a 40-60% reduction in token usage compared to default settings.
